# Решение 1
**master-slave** - есть один главный сервер и один или несколько подчинённых.
В этой конфигурации:
   - *Master (ведущий)* — основной сервер. Все операции записи (INSERT,UPDATE, DELETE) происходят только на нём.
   - *Slave (ведомый)* — копия мастера. Он получает все изменения от мастера и обслуживает запросы на чтение (SELECT).

**Плюсы**
  - Повышение производительности за счет распределения читающих запросов, master в основном занимается записью
  - Данные дублируются, при выходе из строя master один из slave может стать новым мастером

**Минусы**
  - Данные на slave появляются с задержкой
  - Все изменения только через master
  - Чтобы slave стал master требуется ручная настройка
  - При выходе из строя master возможна утеря данных, если они не успели синхронизироваться
    
**master-master** - все серверы равноправны — каждый является одновременно и мастером, и слейвом для другого. Запросы на запись или чтение можно отправлять на любой из серверов, и изменения будут распространены на все остальные.
